# Shell 公共能力升级准入规范

## 版本

- v1.1.0 (2026-07-03)
- 适用范围：Shell-1 及后续 Shell 系列

## 1. 目标

本规范用于回答一个问题：某项能力是继续放在游戏内实现，还是上升为 Shell 公共能力。

核心目标：

1. 避免重复编程，降低长期维护成本。
2. 避免过度抽象，防止 Shell 被单游戏需求污染。
3. 保障向后兼容，不影响已上线模块稳定运行。
4. 形成标准化流程，做到可评审、可追溯、可回滚。

## 2. 升级判定标准（全部满足）

候选能力必须同时满足以下 5 条：

1. 复用范围：至少 2-3 个游戏已出现相同需求。
2. 语义稳定：能力定义清晰，短期不会频繁变形。
3. 接口最小：可以抽象为简洁 API，不需要业务分支堆叠。
4. 可开关：默认行为可保持现状，新增行为通过配置启用。
5. 可回归：能列出明确回归清单并通过验证。

## 3. 禁止上升到 Shell 的情况

出现以下任意情况，必须保持在游戏层：

1. 仅服务单一游戏题型或临时活动。
2. 依赖特定素材、特定文案或特定玩法语义。
3. 抽象后反而增加学习成本或配置复杂度。
4. 无法保证旧游戏零改动可运行。
5. 缺少回归方案或无法验证风险边界。

## 4. 标准流程（Gate Process）

### Gate 0：提案登记

提交提案单（见附录 A），说明重复点、收益、风险和回滚方案。

### Gate 1：游戏内试点

先在 1 个游戏内实现并验证，产出可复用证据。

### Gate 2：复用确认

在第 2 个游戏复用，确认不是一次性需求。

### Gate 3：Shell 抽象设计

定义最小 API、默认行为、配置开关、错误处理与边界。

### Gate 4：实现与回归

完成实现并执行回归清单（见第 6 节）。

### Gate 5：发布与观察

按版本发布，观察 1-2 个迭代周期，记录问题并可快速回滚。

## 5. 兼容性与开关策略

1. 默认兼容：未配置时保持历史行为不变。
2. 显式启用：新能力必须通过配置开启，不做静默替换。
3. 渐进迁移：允许新旧路径并存一个版本周期。
4. 明确弃用：若计划移除旧路径，必须提前发布弃用公告。

## 6. 最小回归清单（必须通过）

1. Home / Game / Result 三屏流程正常。
2. 语言切换（zh/en）正常。
3. 语音开关与播报正常。
4. 进度、得分、提示逻辑正常。
5. localStorage key 与历史统计无破坏。
6. 旧游戏零改动运行通过。
7. 移动端（<=520px）关键布局无错位。

## 7. 发布与回滚

### 发布要求

1. 在主公约中登记变更点与适用版本。
2. 在变更说明中提供升级示例与不升级示例。
3. 标记 owner 与观测周期。

### 回滚要求

1. 提供一键关闭开关或等效降级路径。
2. 不做不可逆数据迁移，或提供迁移回退方案。
3. 回滚后旧游戏可继续运行。

## 8. 角色与职责

1. 提案人：提交提案与试点证据。
2. Shell 维护者：评审抽象合理性与兼容性。
3. 游戏负责人：验证真实接入成本和可用性。
4. QA：执行回归清单并记录风险。

## 9. 已完成 Gate 案例记录

### Shell-2：createReasoningGame（2026-07-03）

| Gate | 结论 |
|------|------|
| Gate 0 提案登记 | Logic Pattern 陈述推理型，需要"前提区 + 问题区"替代序列区 |
| Gate 1 游戏内试点 | logic-pattern-hunter 首个验证，比较传递链 12 题全链路跑通 |
| Gate 2 复用确认 | Number Pattern 等未来推理类模块已确认复用此接口 |
| Gate 3 抽象设计 | 独立 HTML 骨架 `_buildReasoningHTML()`；接口缩减为 2 函数 |
| Gate 4 实现与回归 | Shell-1 所有游戏零改动通过；新 CSS 使用 `s2-` 前缀隔离 |
| Gate 5 发布 | v2.2.0 随主公约一同发布；`shell.report()` 增加 `shell` 字段 |

**关键设计决策记录：**
1. Shell-2 作为 `shell.js` 内新入口函数，不新建文件，共享公共层。
2. 前提与问题由 Shell 直接从 `q.premises` / `q.questionZh/En` 自动渲染，游戏无需实现 `renderSequence`。
3. `shell` 字段加入 `report()` 强制标注，保证跨 Shell 数据可归因。
