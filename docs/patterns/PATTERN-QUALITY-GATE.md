# Pattern 模块出题与质量门禁规范

## 版本

- v1.0.0 (2026-07-03)
- 适用范围：所有 Pattern 系列（Number / Color / Visual / Temporal / Motion / Size）

## 1. 目标

本规范用于保证 Pattern 模块在扩展时：

1. 题型边界清晰，可解释且可复核。
2. 干扰项设计一致，错误归因稳定。
3. 数据记录结构统一，可横向比较。
4. 不因题目漂移导致能力标签失真。

## 2. 题型规格必须先行（先规格后实现）

每个新题型上线前必须有题型规格单（见附录 A），至少覆盖：

1. pattern_type
2. ability_tag
3. 允许变量
4. 禁止变量
5. 序列长度
6. 干扰项模板
7. error_type 映射规则
8. 边界策略
9. 示例题与反例题

未完成规格单，不得进入开发。

## 3. 通用质量门禁

### 3.1 单变量原则

单题只允许一个主变量发生变化。若同时变化多个变量，必须拆成独立题型。

### 3.2 可辨识性原则

关键差异必须肉眼可分辨，不允许边界模糊导致多解。

### 3.3 无歧义原则

题目应保证唯一最优答案，且干扰项有明确错误语义。

### 3.4 边界显式原则

越界规则（如最大值后行为）必须在规格单明示，不得隐式处理。

### 3.5 可归因原则

每个错误选项必须可映射到明确 error_type。

## 4. 干扰项与错误类型规范

每题 4 个选项建议结构：

1. correct
2. pattern_misread（规律误读）
3. adjacent_item（相邻项误判）
4. random 或 extreme_out_of_range

循环题可替换为 cycle_misalign。

error_type 命名必须统一使用小写蛇形（snake_case），禁止同义重复命名。

## 5. 数据记录规范

推荐单题记录字段：

```json
{
  "module": "size_pattern",
  "pattern_type": "grow",
  "level": 1,
  "correct": true,
  "time_ms": 3500,
  "error_type": "none"
}
```

要求：

1. module 与 pattern_type 必填。
2. time_ms 使用毫秒整数。
3. error_type 来自预定义集合。
4. ability_tag 映射规则在模块级文档中固定。

## 6. Size Pattern 专项边界（首版固定）

1. 只做纯视觉大小感知。
2. 禁止数字、单位、百分比、倍数计算。
3. 固定离散档位（3-4 档）。
4. 统一基线对齐，避免位置线索干扰。
5. 同题固定同一图形，仅改变整体大小。
6. 首版不做动画，不做二维长宽比变化。

## 7. 上线前最小测试集

### 7.1 出题校验

1. 100 道随机题无越界。
2. 所有题 answer 均包含在 options 内。
3. 所有 options 可映射到已定义 optionTypes。
4. 无重复选项、无空值选项。

### 7.2 渲染校验

1. 问号占位统一为 '?'.
2. 序列与选项在移动端可完整显示。
3. 基线对齐策略在各屏宽下一致。

### 7.3 统计校验

1. 正误计数正确。
2. error_type 统计与作答结果一致。
3. 历史记录写入与读取正常。

## 8. 变更管理

任何题型规则修改必须同步更新：

1. 本文档对应条目。
2. 该模块设计说明（如 Size Pattern 设计文档）。
3. 数据版本说明与变更记录。
